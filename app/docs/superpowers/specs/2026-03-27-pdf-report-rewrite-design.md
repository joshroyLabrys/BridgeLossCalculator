# PDF Report Rewrite — Design Spec

**Date**: 2026-03-27
**Status**: Approved
**Approach**: Replace browser `window.print()` with `@react-pdf/renderer` for deterministic PDF generation

## Problem

The current PDF report uses `window.print()` with CSS media queries. This has caused 4+ major bugs due to:
- Browser print engine inconsistencies (display:none rendering, CSS specificity conflicts)
- No real page-break control (tables split mid-row)
- Fragile chart capture via `beforeprint` event (fires unreliably across browsers)
- SVG charts can spill horizontally with no size guarantees
- Hardcoded row pagination (38 rows/page) breaks with any font/margin change

## Solution

Replace the entire print-based approach with `@react-pdf/renderer`, which generates PDFs in-memory as a direct download. No browser print engine involved.

## Architecture

### Files Changed

| File | Action |
|------|--------|
| `src/components/pdf-report.tsx` | **NEW** — react-pdf Document component + `generatePdf()` function |
| `src/components/pdf-report-charts.ts` | **NEW** — SVG-to-PNG chart capture utility |
| `src/components/print-report.tsx` | **DELETE** — replaced entirely |
| `src/app/print-styles.ts` | **DELETE** — no CSS print rules needed |
| `src/app/layout.tsx` | **EDIT** — remove `dangerouslySetInnerHTML` print style injection |
| `src/components/main-tabs.tsx` | **EDIT** — replace `window.print()` with `generatePdf()` call |

### Generation Flow

1. User clicks "Generate PDF" button
2. `generatePdf()` captures live SVG charts from DOM → converts to PNG data URLs via offscreen canvas at 2x resolution
3. Renders react-pdf `<Document>` with all store data + chart images
4. Creates blob via `pdf().toBlob()`
5. Triggers browser download as `{projectName}-report.pdf`

### Report Page Structure

| Page | Section | Content |
|------|---------|---------|
| 1 | Cover | Title, project name, date, disclaimer |
| 2 | Input Summary | Bridge geometry grid, coefficients grid, flow profiles table, cross-section chart image |
| 3+ | Cross-Section Data | Paginated table with auto-overflow and repeated headers |
| N | Freeboard & Flow Regime | Freeboard check table with color-coded status, flow regime matrix (F/P/O) |
| N+1 | Method Comparison | Four sub-tables: Upstream WSEL, Head Loss, Approach Velocity, Froude Number |
| N+2 | Charts | Afflux rating curve + WSEL chart as embedded PNG images |
| N+3 | AI Analysis | Structured bullet sections (if AI summary exists) |

### Table Pagination Strategy

- Each table row uses `wrap={false}` to prevent mid-row splits
- Tables that overflow a page automatically continue on the next page
- Continued tables repeat the header row on the new page
- Cross-section data uses a custom paginated table component that tracks available space

### Chart Embedding

- SVGs are found via `data-chart-id` attributes (replacing fragile class-name selectors)
- Each SVG is serialized, drawn to an offscreen canvas at 2x device pixel ratio
- Resulting PNG data URL is embedded as react-pdf `<Image>`
- Charts are sized to `width: 100%` with aspect ratio preserved
- If no charts are found (e.g., user hasn't navigated to summary tab), the Charts page is omitted

### Page Template

Every page (except cover) includes:
- **Header**: Report title (left) + section name (right), separated by a thin border
- **Footer**: "Generated by Bridge Loss Calculator - QA verification only" (left) + date + page number (right)
- **Margins**: 15mm all sides (A4)

### Typography

- Font: Helvetica (built into react-pdf, no font embedding needed for v1)
- Body: 10pt
- Table cells: 9pt
- Headers: 12pt semibold
- Section descriptions: 8pt gray

### Color Coding

- Freeboard status: green (clear), amber (low), red (pressure/overtopping)
- Flow regime warnings: amber for Yarnell under non-free-surface conditions
- Method colors in charts match the app: Energy blue, Momentum green, Yarnell orange, WSPRO purple

## Dependencies

- `@react-pdf/renderer` — PDF generation
- No other new dependencies required

## What Gets Removed

- `print-report.tsx` (entire file)
- `print-styles.ts` (entire file)
- `beforeprint` event listener
- `dangerouslySetInnerHTML` style injection in `layout.tsx`
- All `@media print` CSS rules

## Success Criteria

- PDF downloads instantly on button click (no print dialog)
- Tables never split mid-row across pages
- Charts render at correct size, never spill horizontally
- All data from the app appears in the report (no missing data)
- Report looks identical across Chrome, Firefox, Safari, Edge
- Professional enough for engineering team use
